## Step 4: Read the base project research

Read ~/github/sw-embed/sw-cor24-apl/docs/research.txt thoroughly. This is the
comprehensive design document for the APL implementation. Key aspects:
- ASCII surface syntax: lowercase reserved words (iota, rho, take, drop, etc.)
- Uppercase user identifiers (A, FOO, MATRIX1)
- Context-sensitive display: monadic rho = "shape-of", dyadic A rho B = "reshape"
- Editor prettification: display rho as the APL glyph but store canonical ASCII
- Integer-only APL, rank <= 2, small arrays
- Historical APL implementations for context

Also read ~/github/sw-embed/sw-cor24-apl/Cargo.toml and any other docs/ files
to understand the current state of the base APL project.
